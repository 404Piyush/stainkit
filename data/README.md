# Datasets

This directory holds the input and output of every pipeline run.
Three subdirectories are tracked:

* `data/raw/` — input histopathology images. **Not** tracked by git
  (see `.gitignore`); the folder is preserved across clones via
  `data/raw/.gitkeep`.
* `data/processed/` — stain-normalised images and tissue masks.
  Tracked.
* `data/benchmark/` — per-image benchmark CSVs and plots. Tracked.

## Datasets used in the bundled benchmark

| Source                | What we get                                | Size       | License       |
| --------------------- | ------------------------------------------ | ---------- | ------------- |
| PatchCamelyon (PCam)  | 96x96 RGB patches from CAMELYON16/TCGA     | ~80 MB     | research      |
| TCGA (via GDC API)    | Whole-slide SVS images of breast tissue    | 1-3 GB/each| open access   |
| Synthetic (script)    | 128x128 RGB H&E-like images                | <1 MB      | n/a           |

The synthetic dataset is generated by
`scripts/generate_sample_data.py` and is enough to exercise the
pipeline on machines that have no real dataset available.

## How to populate `data/raw/`

```bash
# 100 PatchCamelyon patches -> data/raw/
python scripts/download_pcam.py --num-images 100 --output data/raw

# 5 small TCGA whole-slide images (multi-GB) -> data/raw/
python scripts/download_tcga.py --num-images 5 --output data/raw

# 8 synthetic images -> data/raw/
python scripts/generate_sample_data.py --num-images 8 --output data/raw
```

The TCGA downloader prefers the smallest available SVS files so the
download finishes within a Colab session.
